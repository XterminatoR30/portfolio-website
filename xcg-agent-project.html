Key Features

Automated Daily Briefings

Structured intelligence reports covering 4 market categories (Crypto, Commodities, FX, Indonesian Stocks) plus deep news search across AI, Digital Marketing, IDX Stocks, and Hot Crypto developments. Delivered daily via Telegram with macro correlation analysis.

Multi-Layer Security

Enterprise-grade protection with prompt injection detection (17 attack signatures), rate limiting (5 requests/minute + 5 queries/day per user), NSFW filtering, and admin-only cron authentication. User input wrapped in tags to prevent instruction hijacking.

Intelligent Tool System

4 specialized tools: Tavily web search with news/text modes, aggressive BeautifulSoup web scraper, yfinance real-time market data (13 instruments), and dynamic Indonesian news from 12 financial portals (IDXChannel, Katadata, Kontan, etc.).

RAG Memory System

ChromaDB vector store with OpenAI embeddings for persistent memory. Daily briefings auto-saved for historical analysis. Users query via /ask command with k=3 retrieval. Supports multi-turn conversations with contextual awareness.

Multi-Language Intelligence

Bilingual output: English for AI/Tech, Digital Marketing, Crypto news. Bahasa Indonesia for Indonesian Stocks (IDX) and Economy sections. Search optimization uses mode="text" for local Indonesian sites and domain filtering for financial portals.

Zero Duplication

SQLite deduplication engine tracks unique URLs from briefings. Prevents news re-reporting across sessions. Extracts both HTML href and markdown links. Maintains 15-item history for LLM context. 99.9% uniqueness rate.

System Architecture

Multi-layer autonomous agent with security, tools, and memory components

Layer 1: Telegram Interface & Security Shield

Entry Point: FastAPI webhook receives Telegram messages
Security: Prompt injection scanning (17 forbidden signatures) + NSFW filtering + rate limiting
Rate Limits: 5 requests/minute (IP) + 5 queries/day (chat_id)
Authentication: X-Cron-Secret header for admin endpoints

Layer 2: LangChain Agent Orchestration

Public Agent: User queries wrapped in <user_input> tags to block injection
Cron Agent: Admin tasks wrapped in <admin_task> namespace
Model: GPT-4-turbo-mini (temperature 0.5)
Execution: AgentExecutor with verbose logging

Layer 3: Tool Ecosystem

search_web: Tavily API with news/text modes + time filters
scrape_website: BeautifulSoup with paragraph extraction
get_market_data: yfinance for 13 instruments (BTC, ETH, SOL, Gold, USD/IDR, IHSG, BBCA)
get_indonesian_market_news: Domain-filtered Tavily across 12 portals

Layer 4: Memory & Deduplication

ChromaDB: Vector store with OpenAI embeddings (k=3)
SQLite user_limits: Tracks daily query counts per chat_id
SQLite sent_news: URL-based deduplication
Persistence: ./chroma_db/ + agent_memory.db

Layer 5: Telegram Delivery

Message Splitting: Smart chunking for 4000+ char messages (paragraph-aware)
HTML Mode: <b>, <i>, <a href> tags for formatted output
Broadcast: Comma-separated chat IDs for multi-user delivery
Web Preview: Disabled to keep chat clean
